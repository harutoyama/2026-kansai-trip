# ADR-0004 Supabaseによる匿名共同編集と共有メモフィード

- 状態: 採用
- 決定日: 2026-07-20

## 背景

既存アプリは確定旅程と交通情報の閲覧に強い一方、USJ作戦、食事候補、京都の希望、家族間のメモをWeb上で継続的に更新する領域がなかった。静的Markdownや`src/data/trip.ts`を変更する方式では、技術知識のない家族がスマートフォンから編集できず、再デプロイも必要になる。

## 決定

- 既存旅程の正本と旅程画面は維持する。
- USJ、食事、京都の固定作戦ページを`shared_pages`へ保存する。
- 任意の新規メモを`shared_memos`へ保存し、時系列フィードとして表示する。
- Supabase Postgres Changesを購読し、更新時に各端末が再読込する。
- アカウント登録を要求せず、匿名ロールへ共同編集権限を与える。
- Google Docs型の文字単位同時編集やCRDTは採用しない。

## 理由

家族4人が短期間利用する用途では、投稿単位とページ単位の最終書き込み優先で十分である。CRDTや独自認証を導入するより、操作の簡単さ、実装量、保守負担を優先する。

## 結果

### 利点

- スマートフォンから新規メモを追加できる。
- 投稿後に編集と削除ができる。
- 別端末へ変更が自動反映される。
- 既存の確定旅程を共同編集データから分離できる。

### 欠点とリスク

- URLと公開キーを知る第三者も編集できる。
- 同じページを同時編集した場合は、後から保存した内容が優先される。
- Supabase未設定または障害時は共同編集できない。
- Realtimeイベントの到着順やネットワーク遅延により、一時的に表示差が生じる可能性がある。

## 対策

- 保存可能な文字数とカテゴリーをDB制約で制限する。
- 機微情報を保存しないことを画面と文書で明示する。
- Supabase未設定時は編集操作を無効化し、静的サンプルだけを表示する。
- 旅行終了後は匿名書き込みポリシーを停止する。
