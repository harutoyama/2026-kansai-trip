# ADR-0005 作戦と検討メモを統合表示する計画ハブ

- 状態: 採用
- 決定日: 2026-07-21

## 背景

従来は「作戦」と「共有メモ」が別画面だったが、双方に候補や方針が混在し、利用者から見た責務が不明確だった。また、作戦はUSJ・食事・京都の固定3件のみで、新しい作戦を追加できなかった。

## 決定

- 下部ナビゲーションの「作戦」と「メモ」を「計画」に統合する。
- 計画画面内では「作戦」と「検討メモ」を別セクションとして表示する。
- 作戦は、目的・優先順位・手順・代替案をまとめた実行方針とする。
- 検討メモは、候補・調査結果・確認事項・ToDoとする。
- 作戦には検討中・採用中・予備・終了の状態を持たせる。
- 検討メモには未対応・確認済み・作戦へ反映済み・不要の状態を持たせる。
- 既存の固定3ページは削除不能な基本作戦として維持する。
- 追加作戦と検討メモは、既存の`shared_memos`テーブルに後方互換な識別情報を付けて保存する。
- 既存の未識別メモは、未対応の候補メモとして読み込む。

## 理由

本番Supabaseに保存済みの共同編集データを維持し、追加のDB移行操作なしで一度のアプリ更新により新UIへ移行するためである。UI上は責務を明確に分けつつ、保存層は現行テーブルと互換にする。

## 結果

### 利点

- 作戦を任意件数追加できる。
- 候補情報と採用済み方針を同じテーマ内で比較できる。
- 既存メモを失わずに移行できる。
- Supabase SQLの追加実行が不要である。

### 制約

- 固定3件の基本作戦は、カテゴリー・タイトル・状態を変更できない。
- 保存層では追加メタデータをタイトルの予約形式として保持する。
- 将来、大規模化する場合は専用の`plans`と`planning_notes`テーブルへ移行する。
